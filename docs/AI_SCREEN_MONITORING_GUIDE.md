## ğŸš¨ **Troubleshooting**

### **"Screenshot not available in cloud environment"**

**Solution:** Screen monitoring requires running VATSAL locally on your desktop (Windows/Mac/Linux). It cannot work in cloud/browser environments due to security restrictions.

### **Too many API calls / slow performance**

**Solutions:**
1. Increase check interval (60s â†’ 120s)
2. Enable Change Detection (skips identical screens)
3. Reduce number of active modes (use 2-3 instead of all 8)
4. Use Smart Scheduling

### **Analytics not updating**

**Check:**
1. Monitoring is actually running (not paused)
2. Screen is changing (change detection needs variation)
3. Analysis modes are selected correctly
4. No errors in output console

### **Alerts not triggering**

**Check:**
1. Triggers are set correctly (âš™ï¸ Configure Settings)
2. Auto Actions are enabled
3. Monitoring is running (not paused)
4. Conditions are being met (check analytics)
